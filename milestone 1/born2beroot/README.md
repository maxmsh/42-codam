Born2BeRoot is a simple exercise in setting up your own virtual machine and configuring system settings, such as user privileges and password security features. This was a no-code exercise and thus only the signature of my virtual machine was submitted.

There are a few domains one should be informed about when setting up his or her own virtual machine in Born2BeRoot, such as:

## Operating System ##

To start this exercise off with, one should decide what operating system they would like to install on their virtual machine. The subject mentions both Debian and Rocky as examples. Since Debian is much easier to manage for beginners due to the large amount of documentation all over the web and people who actively maintain such sources, I went with Debian over Rocky. While I thoroughly enjoyed this exercise, I would not say my skills are at a sufficient enough level to be classified as advanced yet. Besides, I had other bases to cover.

## Package Managers ##

Package managers like apt and aptitude are used to search and/or (de)install packages from the internet.  

## User Permissions ##

An important part of Born2BeRoot is managing user permissions effectively. It may happen that when you start your virtual machine, sudo is not preinstalled and you'll have to log in as root to gain access to superuser permissions.
